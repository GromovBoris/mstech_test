# Test Task for the Position of HTML Layout Designer

## Part 1

You can find the completed Figma layout in the /docs folder.

Minor deviations from the original task may occur due to generation factors and inaccuracies.

### Requirements for the Task:

1. The layout must be responsive with a minimum display resolution of 320px. Responsiveness is at your discretion; however, the page must display correctly in the browser at any resolution.

2. All resources in the layout must be connected locally.

3. The layout must be flexible: when changing the amount of text, list items, or form fields, the layout should not break.

4. The form must be ready for operation (each field should be input[name='...']).

5. The header should stick to the top of the page when scrolling, with a darkened background.

6. The use of libraries, HTML/CSS frameworks, or grids like Bootstrap and others is not allowed.

7. Initiative in creating dynamics on the page (animations, hover effects) is encouraged.

8. All textual and graphic content must be replaced with generated content (for example, using GPT and Ideogram). The theme of the content is up to you, and the language is English.

9. The final project should consist only of regular HTML/CSS/JS files (without preprocessors and build tools).

10. The completed test task must be uploaded to GitHub with a page on GitHub Pages, Netlify, or Vercel.

11. The test task should be submitted with a link to the deployment + a link to GitHub.
